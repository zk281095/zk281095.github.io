定义变量未使用

开发过程中经常遇到定义了变量，但是未使用并且忘记注释或删除的时候，产生报错。
这是golang的一个很好的开发机制，应该养成这样的习惯，但是个人觉得在开发过程中这样会有可能降低效率。

var x string
_ = x

通过这种形式在文件开头对定义的变量进行复制，屏蔽系统的这个功能，然后在开发完成之后删除_ = x，不失为一种方式
